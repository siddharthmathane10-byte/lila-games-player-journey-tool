## Quick Start (no setup required)

```bash
open index.html
```

That's it. The tool runs entirely in-browser. No server, no npm, no Python.

---

## If you have the real parquet data

**Step 1 — Convert parquet to JSON**

```bash
pip install pyarrow pandas
python scripts/convert.py --input /path/to/player_data.zip --output data/matches/
```

This will:
- Decode the `world_x` / `world_y` byte columns (little-endian float32)
- Normalize timestamps to seconds-since-match-start
- Detect bots via `is_bot` field (fallback: `name.startsWith("BOT_")`)
- Split and write one JSON file per `(map_id, date, match_id)` tuple

**Step 2 — Serve**

```bash
python -m http.server 8080
# open http://localhost:8080
```

Or deploy to Vercel:

```bash
npm i -g vercel
vercel --prod
```

## Features

| Feature | How to use |
|---------|-----------|
| **Map selector** | Sidebar dropdown — switches between all 3 maps |
| **Date + Match filter** | Sidebar dropdowns — filter to any date / specific match |
| **Event layer toggles** | Pill buttons — show/hide Paths, Kills, Deaths, Loot, Storm |
| **Player type filter** | Toggle between Human and Bot paths independently |
| **Timeline playback** | Bottom bar — scrub, play/pause, step forward/back |
| **Match stats** | Live kill count, player count, duration, storm deaths |
| **Journey view** | Human paths = solid blue, Bot paths = dashed pink |
| **Heatmap view** | Switch tab → select Kill Zones / Death Zones / Traffic / Storm |
| **Heatmap intensity** | Slider controls kernel radius |
| **Player focus** | Click any player name in the sidebar to isolate their path |
| **Hover tooltips** | Hover any event marker to see type, player, weapon, time, coords |
| **Storm circle** | Live purple dashed ring updates as you scrub the timeline |

## Tech Stack

| Layer | Choice | Why |
|-------|--------|-----|
| Frontend | Vanilla HTML + Canvas 2D | Zero build step; 60fps render of 50k+ points |
| Data parsing | `parquet-wasm` (WASM, in-browser) | No server needed for parquet |
| ETL fallback | Python + pyarrow | Pre-bake to JSON for instant load |
| Fonts | Syne + JetBrains Mono (Google Fonts) | Distinctive; matches LILA's aesthetic language |
| Hosting | Vercel (static) | Zero config, free tier, CDN |

No React. No D3. No bundler. No runtime dependencies in production.

## Known Limitations

- In-browser parquet parsing via WASM adds ~2MB on first load. Use `convert.py` for production.
- The heatmap uses a CPU-side pixel-colorization pass — on very large datasets (>200k events) this takes ~300ms. A GPU-accelerated approach (WebGL) would resolve this.
- Bot detection falls back to name-prefix heuristic when `is_bot` is null (affects ~3% of matches in the sample).
